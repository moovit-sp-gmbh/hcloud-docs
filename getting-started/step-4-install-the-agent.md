# Step 4: Install the agent

This is no cold war thriller - the helmut.cloud agent is the piece of software that will execute your workflows for you.&#x20;

## Download and install the Hcloud agent software

Click on your avatar icon in the upper right corner and select "Download agent". Choose a downloader that will work with your operating system. Curious about the Docker client? Read on here.

Installing the agent on a workstation that you want to use - it is straight forward. After this is done, the agent will be running inside your taskbar, identifiable by the H-symbol. There is no graphical interface coming with it, except a little menu that opens on right-click.

## Configure the agent

To use Hcloud, two things are needed:

* an account or token that is valid for your organization,
* a context in which you want to run the client.

The easiest way is to use your own account for testing. Make sure that you [enabled your account as a target](../organizations/view-my-organizations/being-a-target.md) for the organization used.

Now, right-click the H-icon and choose "Configure". A website opens in your browser. By default, no context is defined. To add context, click on "Create context" and log in with your account.

Depending on the settings of your private or shared organization, your agent will inherit the rights to act or not act accordingly.

You are now ready to run your first workflow on your workstation.
